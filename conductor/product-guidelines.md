# Branding
Trigger Move Windows should feel like a native feature of the GNOME Shell. The branding should be subtle, focusing on functionality rather than individual identity.

# UX Principles
- **Efficiency:** The extension's primary purpose is to save time. All interactions should be as fast and direct as possible.
- **Predictability:** Windows should move exactly where the user expects based on their configuration.
- **Minimalism:** No unnecessary UI elements. The extension should stay out of the user's way until triggered.
- **Transparency:** Provide clear visual feedback (notifications) when actions are taken, and detailed logs for troubleshooting.

# Prose Style
- **Tone:** Professional, direct, and helpful.
- **Language:** Use standard GNOME terminology (e.g., "Workspaces", "Activities Overview", "Dash").
- **Instructions:** Clear and concise, avoiding jargon where possible.

# UI Guidelines
- **Preferences UI:** Adhere to the `libadwaita` design patterns for GNOME 45+. Use standard widgets like `Adw.PreferencesPage`, `Adw.PreferencesGroup`, and `Adw.ActionRow`.
- **Icons:** Use standard symbolic icons from the GNOME icon library where appropriate.
- **Visual Feedback:** Use standard GNOME desktop notifications for status updates.
